<decisions_made>

- **Separator cleanup approach**: Clean doubled separators (`..` -> `.`), leading separators (`.pr123` -> `pr123`), and trailing separators (`pr123.` -> `pr123`). Applies to `.`, `-`, `_` characters. Chosen over just-pass-branchName because it's a defensive safety net.
- **No hardcoded fallback patterns in extractPrNumber**: Use configured pattern (or default pattern if no config) as source of truth, then fall back to `gh` CLI by branch name. Removes the divergent hardcoded pattern lists.
- **Unified extractPrNumber location**: `src/lib/worktree-utils.ts` — shared by both lswt and cleanpr.
- **Auto-setup detection**: Implicit — detects when resolved `worktreeParent` path is inside repo root. No new config flag needed.
- **Auto-setup confirmation**: Interactive prompt unless `--json` or `--yes` flags are set. One prompt covering both mkdir and gitignore update.
- **gh CLI fallback**: Uses existing `github.getPrByBranch()` function. Looks up branch from `git.listWorktrees()` first.

</decisions_made>

<context>
This work addresses three user-reported issues with the config system when used globally:

1. The `{slug}` placeholder in `worktreePattern` produces broken paths (e.g., `pr123.`) because `branchName` is never passed to `generateWorktreePath()` at 7 of 8 call sites — even though it's always in scope.

2. Two different `extractPrNumber()` implementations exist (lswt uses hardcoded patterns, cleanpr uses config-aware + hardcoded fallback). Neither handles cases where the naming pattern doesn't match standard patterns.

3. Setting `worktreeParent: ".worktrees"` globally requires manual directory creation and gitignore setup in each repo.

The implementation plan follows strict TDD — write failing tests first, then implement. Each change is independent enough to be committed separately. The plan document has exact code snippets, file paths, and line numbers for every change.
